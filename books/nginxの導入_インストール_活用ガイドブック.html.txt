<h1 id="nginxの入手方法と前提条件" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">nginxの入手方法と前提条件</h1>

<h3 id="入手元・ダウンロード先" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">入手元・ダウンロード先</h3>

Windows版nginxは、インストール版ではなく、実行ファイルを含むZIP形式で提供されます。<br/><br/>

1. 公式ダウンロードページ（https://nginx.org/en/download.html）へアクセスします。
2. **"Mainline version"**（最新機能）または **"Stable version"**（安定版）を選択します。通常は最新のMainline版（例：`nginx/Windows-1.29.5`）を推奨します。
3. `nginx/Windows-x.xx.x` というリンクをクリックし、ZIPファイルをダウンロードします。<br/><br/>

<h3 id="事前確認事項" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">事前確認事項</h3>

nginxはデフォルトでポート **80** を使用します。起動前に、ポート80が他のアプリケーションで使われていないか確認しましょう。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# ポート80を使用しているプロセスを確認
netstat -ano | findstr :80

# 出力例:
# （何も表示されない場合は空いています）
</code>
</pre>
</div>

右端の数字がPID（プロセスID）です。PIDが `4` の場合は「System」プロセス（IISなど）の可能性があります。競合が確認された場合は、後述のnginx.confでポートを変更して対処します。<br/><br/>

- <b><b>***OS バージョン***</b></b>：Windows 11（64bit推奨）
- <b><b>***管理者権限***</b></b>：インストール（展開）先のフォルダによっては必要です。
- <b><b>***ポートの競合***</b></b>：デフォルトでポート **80** を使用します。IISやApacheが動作している場合は停止するか設定変更が必要です。<br/><br/>

<h1 id="nginxのインストール・セットアップ手順" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">nginxのインストール・セットアップ手順</h1>

<h3 id="手動インストールと起動" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">手動インストールと起動（推奨）</h3>

<b><b>***ステップ 1：ファイルの展開***</b></b><br/><br/>
1. ダウンロードしたZIPファイルを右クリックし、「すべて展開」を選択します。
2. 展開先として `C:\nginx` などのシンプルなパスを指定することを推奨します。<br/><br/>

> 注意：パスに日本語・スペースが含まれないようにしてください（例：`C:\Users\山田\nginx` は不可）。`C:\nginx` が最もシンプルで推奨です。<br/><br/>

展開後のフォルダ構成は以下のようになります：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
C:\nginx\
├── nginx.exe              ← nginxの本体（実行ファイル）
├── conf\
│   └── nginx.conf         ← メイン設定ファイル（ここを編集する）
├── html\
│   ├── index.html         ← デフォルトのトップページ
│   └── 50x.html           ← エラーページ
├── logs\
│   ├── access.log         ← アクセスログ
│   └── error.log          ← エラーログ
└── temp\                  ← 一時ファイル置き場
</code>
</pre>
</div>

<b><b>***ステップ 2：基本設定の変更***</b></b><br/><br/>
ポート80が使用中の場合、`C:\nginx\conf\nginx.conf` を開き、`listen 80;` の行を別のポートに変更します。<br/><br/>

> 注意：設定ファイル（nginx.conf）を編集する際は、BOMなしUTF-8で保存できるエディタ（VS Codeなど）を推奨します。Windowsのメモ帳（Notepad）だと環境によってBOM付きUTF-8で保存されてしまい、nginxが設定ファイルを読み込めなくなるケースがあります。<br/><br/>

<b><b>***ステップ 3：起動と動作確認***</b></b><br/><br/>
**PowerShell** を管理者権限で開き、以下のコマンドを実行します。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# nginxのディレクトリへ移動
cd C:\nginx

# nginx の設定ファイルに文法エラーがないか確認
.\nginx -t

# 出力例:
# nginx: the configuration file C:\nginx/conf/nginx.conf syntax is ok
# nginx: configuration file C:\nginx/conf/nginx.conf test is successful

# nginx を起動
start nginx
</code>
</pre>
</div>

> 注意：必ず `start nginx` を使いましょう。直接 `.\nginx` とだけ打つとPowerShellウィンドウが操作不能になることがあります。<br/><br/>

> 注意2<br/>
start nginx：実行した時に以下のエラーが出た場合は、まず「管理者権限」で実行しているか確認し、それでもだめなら、エクスプローラーからnginx.exeのプロバティを確認し、一番下の「セキュリティ:このファイルは他のコンピューターから取得したものです。このコンピューターを保護するため、このファイルへのアクセスはブロックされる可能性があります。」の隣にある「許可する」チェックボックスがOFFの場合はONにして下さい。<br/><br/>
<div style="width:100%;overflow:scroll;">
<pre style="color:Red;">
<code>

start : このコマンドは、次のエラーのため実行できません: この操作はユーザーによって取り消されました。。
発生場所 行:1 文字:1
+ start nginx
+ ~~~~~~~~~~~
    + CategoryInfo          : InvalidOperation: (:) [Start-Process]、InvalidOperationException
    + FullyQualifiedErrorId : InvalidOperationException,Microsoft.PowerShell.Commands.StartProcessCommand
</code>
</pre>

<b><b>***ステップ 4：ブラウザで動作確認***</b></b><br/><br/>
ブラウザのアドレスバーに以下を入力します：
- ポート80の場合：`http://localhost/`
- ポート変更後の場合：`http://localhost:8080/`<br/><br/>

<b><b>***ステップ 5：管理コマンド一覧***</b></b><br/><br/>
<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# 設定ファイルの文法チェック
.\nginx -t

# 設定を反映（再読み込み）
.\nginx -s reload

# 即時終了
.\nginx -s stop

# 正常終了
.\nginx -s quit
</code>
</pre>
</div>

<h1 id="nginx設定ファイル-nginxconf-の読み方" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">nginx設定ファイル（nginx.conf）の読み方</h1>

<h3 id="設定ファイルの基本構造" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">設定ファイルの基本構造</h3>

nginx.conf は「ブロック」という単位で構成されています。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# グローバル設定
worker_processes  1;

events {
    worker_connections  1024;
}

http {
    server {
        listen       80;
        server_name  localhost;

        location / {
            root   html;
            index  index.html;
        }
    }
}
</code>
</pre>
</div>

<h3 id="シーン1：corsエラーを解決する" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">シーン1：JSONデータを読み込むHTMLページのCORSエラーを解決する</h3>

`index.html` をダブルクリックで開くと発生するCORSエラーは、nginx 経由でアクセスすることで解決できます。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# nginx.conf の修正
location / {
    root   C:/Users/MyUser/Documents/my-project;
    index  index.html;
}

# 設定のリロード
.\nginx -s reload
</code>
</pre>
</div>

<h3 id="シーン3：フロント＋バックエンド連携" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">シーン3：AIエージェント開発環境の構築（フロント＋バックエンド連携）</h3>

`/api/` へのリクエストをバックエンドAPIに転送するリバースプロキシ設定により、同一オリジンとして通信可能です。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
location /api/ {
    proxy_pass         http://localhost:3000/;
    proxy_set_header   Host $host;
    proxy_read_timeout 120s;
}
</code>
</pre>
</div>

<h1 id="windows起動時の自動起動設定" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">Windows起動時の自動起動設定</h1>

タスクスケジューラを使って、PC起動時に自動で nginx を起動できます。<br/><br/>

1. **タスクの作成**：名前を「nginx 自動起動」とする。
2. **全般**：「ユーザーがログオンしているかどうかにかかわらず実行する」を選択。
3. **トリガー**：新規作成で「スタートアップ時」を選択。
4. **操作**：「プログラムの開始」で `C:\nginx\nginx.exe` を指定し、開始オプションに `C:\nginx` を入力。<br/><br/>

<h3 id="q1-ポート競合エラー" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q1. ポート競合で起動しない</h3>
A. `nginx.conf` で `listen 8080;` のように空いているポートに変更してください。<br/><br/>

<h3 id="q2-設定が反映されない" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q2. 設定が反映されない</h3>
A. `.\nginx -t` で文法ミスがないか確認し、`.\nginx -s reload` を実行してください。<br/><br/>

<h1 id="なぜ-nginx-はインストーラーを用意しないのか" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">なぜ nginx は「インストーラー」を用意しないのか？</h1>

有名なソフトなのに、なぜいまだに <code>ZIP</code> 展開の <code>exe</code> 直叩きという「硬派」なスタイルを貫いているのか。そこにはサーバー向けソフトウェア特有の文化があります。<br/><br/>

<h3 id="理由1-ポータビリティの重視" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">1. 「ポータビリティ（移植性）」の重視</h3>

nginx は、<b><b>***Windows レジストリを一切汚さない***</b></b>設計になっています（本ドキュメントのQ7「アンインストール」の項目からもわかるとおり、フォルダを削除するだけで完全にアンインストール完了です）。<br/><br/>

<div style="overflow-x: auto; -webkit-overflow-scrolling: touch; border-radius: 12px; box-shadow: 0 4px 15px rgba(0,0,0,0.1); margin: 20px 0; border: 1px solid #e0e0e0;"><table style="width: 100%; border-collapse: collapse; font-family: sans-serif; min-width: 600px;"><thead><tr style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white;"><th style="padding: 15px; text-align: left; font-weight: 600; border-bottom: 2px solid rgba(255,255,255,0.2);">方式</th><th style="padding: 15px; text-align: left; font-weight: 600; border-bottom: 2px solid rgba(255,255,255,0.2);">特徴</th></tr></thead><tbody><tr style="background-color: #f9f9f9; transition: background-color 0.3s;" onmouseover="this.style.backgroundColor='#f0fcfb'" onmouseout="this.style.backgroundColor='#f9f9f9'"><td style="padding: 12px 15px; border-bottom: 1px solid #eee; color: #444; line-height: 1.6;"><b>インストーラー（.msiなど）</b></td><td style="padding: 12px 15px; border-bottom: 1px solid #eee; color: #444; line-height: 1.6;">OSの深い場所に設定を書き込み、アンインストールもOS経由で行う</td></tr><tr style="background-color: #ffffff; transition: background-color 0.3s;" onmouseover="this.style.backgroundColor='#f0fcfb'" onmouseout="this.style.backgroundColor='#ffffff'"><td style="padding: 12px 15px; border-bottom: 1px solid #eee; color: #444; line-height: 1.6;"><b>nginx 方式</b></td><td style="padding: 12px 15px; border-bottom: 1px solid #eee; color: #444; line-height: 1.6;">フォルダを移動したり、まるごとコピーしたりするだけで別のサーバーへ引っ越しが可能</td></tr></tbody></table></div>

この「シンプルさ」が、インフラエンジニアには好まれます。<br/><br/>

<h3 id="理由2-サーバー管理の標準的な作法" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">2. 「サーバー管理」の標準的な作法</h3>

サーバー用のミドルウェア（Apache, Redis, PostgreSQL のバイナリ版など）の世界では、特に Linux/Unix 環境において「インストーラーを使わずにバイナリを直接配置する」ことが標準的な作法となっているケースが多いです。<br/><br/>

- 複数のバージョンを同じサーバー内で共存させたり、特定のディレクトリ配下だけで動作させたりする制御がしやすいためです。<br/><br/>
- パッケージマネージャー（<code>apt</code>, <code>yum</code> など）が使えない環境や、バージョンを厳密に固定したい本番環境では、バイナリ直接配置が特に重宝されます。<br/><br/>

<h3 id="理由3-windows版はメインではない" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">3. Windows 版は「メインではない」という側面も</h3>

nginx はもともと Linux/Unix 環境で最大のパフォーマンスを発揮するように設計されています。Windows 版の位置付けについては、公式が明示的に認めています。<br/><br/>

> 公式 GitHub（nginx/nginx）の記述：<br/>
> "Note that the current implementation of NGINX for Windows is at the <b>Proof-of-Concept</b> stage and should only be used for <b>development and testing purposes</b>."<br/>
>（意訳：Windows版の現在の実装は概念実証段階であり、開発・テスト目的のみでの使用が推奨されます）<br/><br/>

さらに、公式の Windows 向けドキュメント（nginx.org）でも「<b>ベータバージョン</b>」と明記されており、接続処理方式が <code>select()</code> / <code>poll()</code> のみに限定されているため、Linux 版と比較して高いパフォーマンスや拡張性は期待できません。<br/><br/>

このような位置付けから、インストーラーを開発・維持するコストをかけるよりも、軽量なバイナリ（ZIP）提供にとどめているという背景があります。<br/><br/>

> <b>まとめると</b>：nginx の ZIP 配布は「手抜き」ではなく、レジストリを汚さないポータブル設計・サーバー業界の文化・Windows 版の開発用途という3つの理由が重なった、合理的な選択です。<br/><br/>

本ガイドブックでは、<b><b>***nginx***</b></b> の導入から活用までを解説しました。Windows環境でのセットアップは非常にシンプルで、強力な開発環境を提供します。<br/><br/>
