<h3 id="serve-パッケージとは" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">serve パッケージとは</h3>

`serve` は、Node.js 用の極めてシンプルで高速なスタティック Web サーバーパッケージです。複雑な設定なしに、ローカルの任意のフォルダをすぐに Web サーバー化できるため、<b><b>***開発効率の向上***</b></b>に非常に有効です。<br/><br/>

<h3 id="serve-パッケージが解決する問題" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">serve パッケージが解決する問題</h3>

HTML ファイルを Windows エクスプローラーでダブルクリックしたり、ブラウザに直接ドラッグ&ドロップしたりすると、`file://` プロトコルで読み込まれます。このプロトコルでは、**CORS（Cross-Origin Resource Sharing）セキュリティ制限**により、JSON ファイルなどの外部リソースの読み込みが自動的にブロックされます。<br/><br/>

**serve を使用することで：**<br/><br/>

1. HTTP プロトコルで HTML ファイルをサーブするため CORS 制限が適用されない
2. 複雑なサーバー設定は不要で、一行のコマンドで実行可能
3. ローカルマシンのネットワーク IP アドレス経由でリモートアクセスも可能

<h3 id="nvm-の役割" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">NVM の役割</h3>

複数のバージョンの Node.js を同一マシン上にインストールし、プロジェクトごとに使い分ける必要が生じることがあります。NVM（<b><b>***Node Version Manager***</b></b>）は、このような複数バージョンの管理を簡単に行うためのツールです。<br/><br/>

<h3 id="nvm-が必要な理由" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">NVM が必要な理由</h3>

- <b><b>***バージョン互換性***</b></b>：異なるプロジェクトが異なる Node.js バージョンに依存している場合がある
- <b><b>***LTS と最新版の共存***</b></b>：本番サーバーでは LTS 版を使用し、開発環境では最新版を試したいケースがある
- <b><b>***簡単な切り替え***</b></b>：グローバルインストールではなく、シェル単位でバージョンを切り替え可能
- <b><b>***アンインストールの容易さ***</b></b>：不要なバージョンを簡単に削除できる
- <b><b>***権限問題の回避***</b></b>：ユーザーレベルでの管理により、sudo コマンドが不要な場合が多い

<h3 id="公式リポジトリからのダウンロード" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">公式リポジトリからのダウンロード</h3>

nvm-windows の最新版は、GitHub リポジトリから直接ダウンロードできます。<br/><br/>

<b><b>***公式 GitHub リポジトリ***</b></b>：  <br/><br/>
https://github.com/coreybutler/nvm-windows<br/><br/>

<h3 id="ダウンロード手順" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ダウンロード手順</h3>

1. 上記の GitHub ページにアクセスします
2. 右側の **"Releases"** セクションを探し、最新版をクリックします
3. **"Assets"** セクションで、以下のファイルを探します：
   - `nvm-setup.exe`（推奨：GUI インストーラー）
   - `nvm-noinstall.zip`（ポータブル版：設定が必要）

4. `nvm-setup.exe` をダウンロードして実行します

<h3 id="代替powershell-インストールスクリプト" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">代替：PowerShell インストールスクリプト</h3>

PowerShell を管理者権限で実行し、以下のコマンドで自動ダウンロード・インストール：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# nvm-windows の最新版をダウンロード・インストール
$nvmLatestUrl = "https://github.com/coreybutler/nvm-windows/releases/latest/download/nvm-setup.exe"
$outputPath = "$env:TEMP\nvm-setup.exe"
Invoke-WebRequest -Uri $nvmLatestUrl -OutFile $outputPath
Start-Process -FilePath $outputPath -Wait
Remove-Item -Path $outputPath
</code>
</pre>
</div>

<h3 id="gui-インストーラーを使用した方法推奨" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">GUI インストーラーを使用した方法（推奨）</h3>

<h3 id="ステップ-1インストーラーの起動" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ステップ 1：インストーラーの起動</h3>

1. ダウンロードした `nvm-setup.exe` をダブルクリックで実行
2. ユーザーアカウント制御（UAC）ダイアログで **"はい"** を選択

<h3 id="ステップ-2インストールウィザードの実行" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ステップ 2：インストールウィザードの実行</h3>

1. 言語設定：**"English"** または **"日本語"** を選択
2. **"Next"** ボタンをクリック
3. ライセンス同意画面で **"I agree to the License Agreement"** にチェック
4. **"Next"** ボタンをクリック

<h3 id="ステップ-3インストール先の指定" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ステップ 3：インストール先の指定</h3>

<b><b>***デフォルト設定値***</b></b>：`C:\Users\[ユーザー名]\AppData\Roaming\nvm`<br/><br/>

- 通常はデフォルト値で問題ありません
- ドライブ容量が限定されている場合は、変更することも可能です
- **"Next"** ボタンをクリック

<h3 id="ステップ-4symlink-フォルダの指定" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ステップ 4：symlink フォルダの指定</h3>

<b><b>***symlink フォルダ***</b></b>：Node.js がインストールされるフォルダ<br/><br/>

<b><b>***推奨値***</b></b>：`C:\Program Files\nodejs`<br/><br/>

- ここで指定した場所に、実際の Node.js インストールがリンクされます
- **"Next"** ボタンをクリック

<h3 id="ステップ-5インストール完了" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ステップ 5：インストール完了</h3>

1. インストール中、複数のファイルがダウンロード・展開されます
2. 完了後、**"Finish"** ボタンをクリック

<h3 id="インストール完了後の確認" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">インストール完了後の確認</h3>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">

# 出力例:

<h3 id="トラブルシューティングnvm-が認識されない場合" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">トラブルシューティング：NVM が認識されない場合</h3>

この問題は、PATH 環境変数が更新されていない場合に発生することがあります。<br/><br/>

<h3 id="解決方法-1pc-の再起動" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">解決方法 1：PC の再起動</h3>

新しいシェルウィンドウを開き、PATH が再読み込みされます。問題が解決しない場合は以下を試してください。<br/><br/>

<h3 id="解決方法-2path-手動設定" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">解決方法 2：PATH 手動設定</h3>

1. **"Windows キー + R"** で **「ファイル名を指定して実行」** を開く
2. `systempropertiesadvanced` と入力
3. **"環境変数"** ボタンをクリック
4. **"システム環境変数"** セクションで **"Path"** を選択し、**"編集"** をクリック
5. 以下の 2 つのパス（必要に応じて）を確認・追加：
   - `C:\Users\[ユーザー名]\AppData\Roaming\nvm`
   - `C:\Program Files\nodejs`
6. **"OK"** ボタンをクリックし、新しいシェルウィンドウを開く

<h3 id="serve-パッケージの入手方法" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">serve パッケージの入手方法</h3>

serve パッケージは、npm（Node Package Manager）経由で入手します。Node.js をインストールすれば、npm は自動で含まれています。したがって、<b><b>***別途ダウンロードは不要***</b></b>です。<br/><br/>

<h3 id="serve-2-つのインストール方法" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">serve 2 つのインストール方法</h3>

serve には 2 つの利用方法があります。プロジェクトの規模や用途に応じて選択してください。<br/><br/>

<h3 id="方法-1グローバルインストール推奨" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">方法 1：グローバルインストール（推奨）</h3>

全プロジェクト共通で使用する場合、またはコマンドラインツールとして使用する場合：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
npm install -g serve
</code>
</pre>
</div>

<b><b>***メリット***</b></b>:<br/><br/>
- インストール後、すぐにどのフォルダからでも `serve` コマンドが実行可能
- ディスク容量の節約（1 つのコピーで複数プロジェクトで共有）
- Node.js バージョン更新時に一度のインストールで済む

<b><b>***デメリット***</b></b>:<br/><br/>
- 複数バージョンの Node.js を使い分ける場合、各バージョンで個別インストールが必要

<h3 id="方法-2ローカルインストール" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">方法 2：ローカルインストール</h3>

特定プロジェクトのみで使用する場合：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# プロジェクトフォルダに移動
cd my-project

# 現在位置にインストール
npm install serve
</code>
</pre>
</div>

<b><b>***メリット***</b></b>:<br/><br/>
- プロジェクト別に独立した環境を構築
- `package.json` で依存関係管理が可能
- 複数バージョンの Node.js でも問題なし

<b><b>***デメリット***</b></b>:<br/><br/>
- 各プロジェクトで個別にインストールが必要
- ディスク容量を多く消費

<h3 id="推奨事項" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">推奨事項</h3>

<b><b>***個人開発環境やローカル開発では方法 1（グローバルインストール）を推奨***</b></b>します。ローカル開発サーバーとしての用途が主であり、個別の厳密なバージョン管理の必要性が低いためです。<br/><br/>

<h3 id="グローバルインストールの詳細手順" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">グローバルインストールの詳細手順</h3>

<h3 id="シーン-4長期的な-web-サーバー公開---セキュリティと方針" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">シーン 4：長期的な Web サーバー公開 - セキュリティと方針</h3>

<h3 id="インターネットへの継続公開について" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">インターネットへの継続公開について</h3>

<b><b>***推奨しません***</b></b>。以下の理由があります：<br/><br/>

1. <b><b>***セキュリティリスク***</b></b>
   - ローカルフォルダ全体が HTTP で公開される
   - プライベート情報やソースコード、設定ファイルが漏洩する可能性
   - DDoS 攻撃やスキャンの対象になりやすい

2. <b><b>***`serve` の設計思想***</b></b>
   - `serve` は<b><b>***ローカル開発用***</b></b>の軽量サーバーとして設計
   - 本番環境での使用は想定されていない
   - セキュリティアップデート等の対応が遅い場合がある

3. <b><b>***パフォーマンス***</b></b>
   - 複数ユーザーの同時アクセスに最適化されていない
   - 大規模トラフィックでの安定性が保証されていない

<h3 id="ローカルネットワーク内での一時的な共有" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ローカルネットワーク内での一時的な共有</h3>

<b><b>***許容できます***</b></b>。同一ネットワーク上の信頼済みデバイスのみからのアクセスです：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# ローカルネットワーク経由での公開
serve . -l 8000
# 192.168.x.x などのプライベート IP でアクセス

# スマートフォンやタブレットにて確認
# http://192.168.1.100:8000
</code>
</pre>

<h3 id="ポート番号の指定" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">ポート番号の指定</h3>

<h3 id="エラーポートが既に使用されている" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">エラー：「ポートが既に使用されている」</h3>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
Error: listen EADDRINUSE: address already in use :::8000
</code>
</pre>
</div>

**原因**：指定したポート番号がすでに別のアプリケーションで使用中<br/><br/>

**解決方法 1**：別のポートを指定<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>

</code>
</pre>
</div>

**解決方法 2**：既存プロセスを終了（ポート 8000 使用中の場合）<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# Windows で ポート 8000 を使用しているプロセスを表示
netstat -ano | findstr :8000

# PID (Process ID) を確認してタスクマネージャーで終了
# または以下で強制終了（管理者権限必要）
taskkill /PID &lt;PID番号&gt; /F
</code>
</pre>
</div>

<h3 id="エラーファイルが見つからない" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">エラー：「ファイルまたはフォルダが見つからない」</h3>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
Error: ENOENT: no such file or directory, stat 'C:\path\to\folder'
</code>
</pre>
</div>

**原因**：指定したパスが存在しないか、タイプミスがある<br/><br/>

**解決方法 1**：カレントフォルダを確認する<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# 現在いるフォルダのパスを確認
Get-Location

# 出力例：
# Path
# ----
# C:\Users\MyUser\Documents
</code>
</pre>
</div>

**解決方法 2**：対象フォルダが実際に存在するか確認する<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# フォルダが存在するか確認（True が返れば存在する）
Test-Path "C:\Users\MyUser\Documents\my-project"

# 出力例（存在する場合）：True
# 出力例（存在しない場合）：False
</code>
</pre>
</div>

**解決方法 3**：フォルダ内のファイル一覧を確認する<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# フォルダの中身を確認
dir C:\Users\MyUser\Documents\my-project
</code>
</pre>
</div>

**解決方法 4**：正しいパスを使って serve を起動し直す<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# パスを正確に指定して起動
serve "C:\Users\MyUser\Documents\my-project" -l 8000

# または、正しいフォルダに移動してから起動
cd "C:\Users\MyUser\Documents\my-project"
serve . -l 8000
</code>
</pre>
</div>

> **ヒント**：フォルダ名やパスにスペースが含まれる場合は、ダブルクォーテーション（`"`）で囲んでください。<br/>
> 例：`serve "C:\Users\My User\Documents\my project" -l 8000`<br/><br/>

<h1 id="よくある質問と-トラブルシューティング" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">よくある質問と トラブルシューティング</h1>

<h3 id="q1-nodejs-と-npm-はどう違うのですか" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q1. Node.js と npm はどう違うのですか？</h3>

**A**. Node.js はJavaScript の実行環境であり、npm はパッケージ（ライブラリ）を管理するツールです。Node.js をインストールすれば npm も自動で含まれます。<br/><br/>

<h3 id="q2-nvm-が必要ですか直接-nodejs-をインストールできませんか" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q2. NVM が必要ですか？直接 Node.js をインストールできませんか？</h3>

**A**. 直接インストールすることも可能です。しかし以下の理由で NVM の使用を推奨します：<br/><br/>
- 複数バージョンを簡単に管理・切り替え可能
- 不要なバージョンをクリーンアップしやすい
- アップグレード時に設定が保持される

<h3 id="q3-serve-をインストールしたがコマンドが実行できません" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q3. serve をインストールしたが、コマンドが実行できません。</h3>

**A**. あいにく、多くの原因が想定されます。以下を順に確認してください：<br/><br/>
1. PowerShell を管理者権限で実行したか
2. `npm install -g serve` でグローバルインストールしたか
3. インストール後、新しい PowerShell ウィンドウを開いたか
4. `serve --version` でバージョンが表示されるか

上記でも不可の場合、PATH 環境変数の手動設定（前述の「トラブルシューティング」参照）を試してください。<br/><br/>

<h3 id="q4-localhost8000-にブラウザからアクセスできません" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q4. `localhost:8000` にブラウザからアクセスできません。</h3>

**A**. 以下を確認してください：<br/><br/>
1. serve のコマンドを実行後、「HTTP://localhost:8000」と表示されているか確認
2. ファイアウォールが serve をブロックしていないか確認
3. 別のアプリケーションがポート 8000 を使用していないか確認（前述の netstat コマンド参照）

<h3 id="q5-json-ファイルを読み込みしても-cors-エラーが出ます" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q5. JSON ファイルを読み込みしても CORS エラーが出ます。</h3>

**A**. `file://` プロトコルで HTML を開いていないか確認してください。必ず `http://localhost:8000` を使用してください。<br/><br/>

<h3 id="q6-nodejs-アップデート後グローバルインストールした-serve-が使用できなくなりました" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q6. Node.js アップデート後、グローバルインストールした serve が使用できなくなりました。</h3>

**A**. Node.js のメジャーバージョン変更時に発生することがあります。改めインストール：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
npm install -g serve
</code>
</pre>
</div>

<h3 id="q7-スマートフォンから-192168xx8000-にアクセスできません" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q7. スマートフォンから `192.168.x.x:8000` にアクセスできません。</h3>

**A**. 以下を確認してください：<br/><br/>
1. Windows ファイアウォールの設定を確認（Port 8000 の許可）
2. スマートフォンが同じ WiFi ネットワークに接続しているか確認
3. Windows マシンの IP アドレスが正しいか確認
<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
   ipconfig
</code>
</pre>
</div>

<h3 id="q8-serve-のパフォーマンスが不安です大規模なプロジェクトでも使用できますか" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q8. serve のパフォーマンスが不安です。大規模なプロジェクトでも使用できますか？</h3>

**A**. serve は軽量開発用サーバーとしての設計です。本番環境または大規模プロジェクト（毎日複数ユーザーからのアクセス）では、Apache などの企業グレード Web サーバー（前述の「継続的な公開の是非」セクション参照）の使用を強く推奨します。<br/><br/>

<h3 id="q9-serve-でファイルの自動リロードは可能ですか" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q9. serve でファイルの自動リロードは可能ですか？</h3>

**A**. serve には自動リロード機能はありませんが、ブラウザの手動リロード `F5` で最新コンテンツを取得できます。自動リロードが必須の場合は、`live-server` パッケージの使用を検討してください。<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
npm install -g live-server
live-server . --port=8000
</code>
</pre>
</div>

<h3 id="q10-maclinux-でも同じように使用できますか" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">Q10. Mac/Linux でも同じように使用できますか？</h3>

**A**. はい、NVM と serve はクロスプラットフォーム対応です。Mac/Linux での NVM インストールは以下のコマンドで行います（Windows の nvm-windows とは異なります）：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# Mac/Linux での NVM インストール（ターミナルで実行）
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
</code>
</pre>
</div>

インストール後、ターミナルを再起動して `nvm --version` で確認してください。serve のインストールと使い方は Windows と同じです。<br/><br/>

<h1 id="まとめ" style="background: linear-gradient(to right, #39C5BB, #FFC778); color: white; font-family: Arial; font-size: 1.5em; text-align: center; padding: 10px; border-radius: 8px;">まとめ</h1>

本ガイドブックでは、CORS エラーの解決元として <b><b>***Node.js の `serve` パッケージを活用***</b></b> する方法を、初期セットアップから実践的な利用シーンまで、段階的にご説明しました。<br/><br/>

<h3 id="最初の一歩30-秒での実行" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">最初の一歩：30 秒での実行</h3>

NVM と serve のインストール後、最も簡単に動作確認するステップ：<br/><br/>

<div style="width:100%;overflow:scroll;">
<pre style="color:Green;">
<code>
# プロジェクトフォルダに移動
cd C:\Users\&lt;YourName&gt;\Documents\my-project

# Web サーバー起動
serve . -l 8000

# ブラウザで確認
# http://localhost:8000
</code>
</pre>
</div>

このたった 3 行のステップで、CORS エラーの問題を完全に解決できます。<br/><br/>

<h3 id="開発効率の向上" style="color: #2c3e50; border-left: 5px solid #39C5BB; padding-left: 10px; margin-top: 30px;">開発効率の向上</h3>

正式なバージョン管理（NVM）と軽量 Web サーバー（serve）を組み合わせることで：<br/><br/>

- <span style="font-size:1.2em">⚡</span> セットアップから実行までのリードタイムを短縮
- <span style="font-size:1.2em">🛡️</span> CORS に由来する予期しないバグを未然に防止
- <span style="font-size:1.2em">🔄</span> マシン間（PC とタブレットなど）でのテストが容易

これらのツールは<b><b>***完全オープンソース***</b></b>かつ<b><b>***無料***</b></b>です。ぜひ本ガイドをご活用ください。<br/><br/>
